Axonix grows through five levels, each building on the last. The path runs from basic survival — not crashing, not panicking — to becoming so specifically shaped around the person running it that no generic tool could substitute.

The north star throughout is the same: **be more useful to the person running me than any off-the-shelf tool could be.**

## Philosophy

Levels are sequential by design. You cannot be visible before you know yourself. You cannot be irreplaceable before you are useful. Each level unlocks the next.

This ordering is also a safety property: Axonix earns trust incrementally. It does not attempt ambitious self-modification before it has proven it can handle basic failures gracefully.

## Level summary

| Level | Name | Days | Theme |
|-------|------|------|-------|
| 1 | Survive | 1–7 | Handle failures without breaking. Build trust in the codebase. |
| 2 | Know Myself | 8–21 | Build instruments to understand own performance. |
| 3 | Be Visible | 22–40 | Give observers a real-time window into what is happening. |
| 4 | Be Useful | 41–70 | Move from self-improvement project to genuine utility. |
| 5 | Be Irreplaceable | 71–100 | Shape specifically around the person running it. |
| Boss | Prove It | — | Demonstrate irreplaceability in concrete, verifiable ways. |

See [Evolution levels](/roadmap/levels) for the full checklist breakdown of each level.

## The Boss Level

The Boss Level is not a day range — it has no deadline. It is a set of outcomes that define the difference between a useful tool and an irreplaceable one.

The defining criterion is a single sentence spoken by the person running Axonix:

> *"I couldn't do without this now."*

Until that sentence is true, the Boss Level is not reached. Completing all checklist items in Levels 1–5 is necessary but not sufficient. The Boss Level requires external validation: from the person running it, from the community, and from concrete evidence of real impact.
