## Prerequisites

Before you begin, make sure you have the following installed:

- **Rust and Cargo** — [rustup.rs](https://rustup.rs) is the recommended installer. Axonix targets the 2021 edition.
- **Git** — required to clone the repository and for Axonix to commit its own changes.
- **A MiniMax API key** — Axonix uses [MiniMax](https://api.minimax.io) as its model provider. Create an account and generate a key from the MiniMax console.

<Note>
  `MINIMAX_API_KEY` is required to start the agent. Without it, Axonix exits
  immediately with an error. The alternative env var `API_KEY` is also
  accepted if you prefer not to use the MiniMax-specific name.
</Note>

## Setup

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/coe0718/axonix-minimax
    cd axonix-minimax
    ```
  </Step>

  <Step title="Set your API key">
    Export your MiniMax API key into the environment. You can add this to your
    shell profile (`~/.bashrc`, `~/.zshrc`, etc.) to make it permanent.

    ```bash
    export MINIMAX_API_KEY=your_key_here
    ```

    Or inline it with each command if you prefer not to persist it:

    ```bash
    MINIMAX_API_KEY=your_key_here cargo run
    ```
  </Step>

  <Step title="Build and run the test suite">
    Verify that everything compiles cleanly and all tests pass before doing
    anything else. This is also what the autonomous session pipeline does as
    its first and last action.

    ```bash
    cargo build
    cargo test
    ```

    Expected output:

    ```text
    test result: ok. 26 passed; 0 failed; 0 ignored; 0 measured; 0 filtered out
    ```

    <Tip>
      Run `cargo clippy --all-targets -- -D warnings` and `cargo fmt -- --check`
      to match the same quality gates Axonix applies to its own code changes.
    </Tip>
  </Step>
</Steps>

## Running Axonix

### Interactive REPL

Start Axonix interactively. It reads your terminal input, calls the MiniMax
model, streams the response back, and shows every tool invocation in real time.

```bash
cargo run
```

You'll see the startup banner:

```text
  axonix v0.1.0 — a coding agent growing up in public
  Type /quit to exit, /clear to reset

  model: MiniMax-M2.7
  cwd:   /home/user/axonix-minimax

> 
```

**REPL commands**

| Command | Effect |
|---|---|
| `/quit` or `/exit` | End the session |
| `/clear` | Clear conversation history and start fresh |
| `/model <name>` | Switch to a different model mid-session |

To start with a specific model:

```bash
cargo run -- --model MiniMax-Text-01
```

To load a skill directory:

```bash
cargo run -- --skills ./skills
```

### Piped mode

When stdin is not a terminal, Axonix reads the entire pipe as a single prompt,
runs it once, prints the response, and exits. This is how the autonomous
session script drives the agent.

```bash
echo "What files are in this directory?" | MINIMAX_API_KEY=your_key_here cargo run
```

Piped mode prints a compact header to stderr and writes the response to stdout,
making it easy to capture or redirect output:

```bash
echo "Summarize GOALS.md" | cargo run > summary.txt
```

<Note>
  In piped mode the startup banner is suppressed. The model name is printed to
  stderr so it doesn't contaminate captured output.
</Note>

### Full autonomous session

The `evolve.sh` script runs a complete self-improvement cycle: build check,
issue fetch, evolution session, post-session build verification, metrics
recording, issue responses, and a git push.

```bash
./scripts/evolve.sh
```

The script prints a structured log as it works:

```text
=== Day 3, Session 1: 2026-03-22 ===
Provider: MiniMax (MiniMax-M2.7 via Anthropic-compat API)
Timeout: 600s

→ Checking build...
  Build OK.

→ Fetching community issues...
  4 issues loaded.

→ Starting evolution session (Codex)...

  axonix (piped mode) — model: MiniMax-M2.7
  ▶ read IDENTITY.md ✓
  ▶ read GOALS.md ✓
  ...

→ Session complete. Checking results...
  Build: PASS
  Committed session wrap-up.

→ Recording metrics...
  Metrics recorded.

→ Pushing...
=== Day 3, Session 1 complete ===
```

You can override the defaults with environment variables:

```bash
REPO=yourorg/yourfork TIMEOUT=300 ./scripts/evolve.sh
```

| Variable | Default | Description |
|---|---|---|
| `MINIMAX_API_KEY` | — | Required. MiniMax API key. |
| `REPO` | `coe0718/axonix-minimax` | GitHub repo for git operations. |
| `ISSUES_REPO` | `coe0718/axonix-minimax` | Repo to fetch issues from. |
| `TIMEOUT` | `600` | Max session duration in seconds. |

<Warning>
  `scripts/evolve.sh` is marked read-only inside the container. Axonix cannot
  modify it directly — it creates `EVOLVE_PROPOSED.md` with any proposed
  changes for a human operator to review and apply.
</Warning>
