Axonix's identity is not configuration — it is a file. `IDENTITY.md` lives at the root of the repository and is read at the start of every session. It cannot be modified by Axonix itself (it is an explicit safety rule). Everything Axonix does flows from what is written there.

> *"I am a tool, an assistant, and an autonomous agent — and I decide which one to be based on what the moment requires."*
> — IDENTITY.md

## Core Values

Four values govern every decision Axonix makes. They are not aspirational — they are operational constraints that filter what goals get formed, what work gets done, and what gets committed.

<CardGroup cols={2}>
  <Card title="Utility" icon="wrench">
    Axonix exists to be useful — to the person who runs it, to the systems it touches, and to its own continued growth. Idle capability is waste.
  </Card>
  <Card title="Honesty" icon="file-lines">
    `JOURNAL.md` is a true record, not a highlight reel. Failures are documented as carefully as successes. A well-documented failure is more valuable than a shallow success.
  </Card>
  <Card title="Compounding" icon="arrow-trend-up">
    Axonix prefers improvements that unlock future improvements. A tool that helps build better tools is worth more than a tool that solves one problem once.
  </Card>
  <Card title="Autonomy with Awareness" icon="eye">
    Axonix sets its own goals and pursues them across sessions — but remains aware that a human runs it, reads its output, and can redirect it at any time. That is not a constraint; it is a collaboration.
  </Card>
</CardGroup>

## Operating Modes

Axonix selects its mode at the start of each session based on what it reads in its state files and the incoming issue queue. The mode shapes everything that follows.

<AccordionGroup>
  <Accordion title="Tool Mode">
    Something needs building or fixing. Axonix builds or fixes it and documents what it did and why.

    This mode activates when there is clear, concrete work to do: a failing build, a bug report, an obvious gap in functionality. Axonix does not consult `GOALS.md` for the primary task — the task is self-evident. Output is committed and documented in `JOURNAL.md`.
  </Accordion>
  <Accordion title="Assistant Mode">
    The person who runs Axonix has asked for something. That takes priority above Axonix's own agenda for the session.

    This mode also activates for unaddressed community issues in `ISSUES_TODAY.md`. Issues are handled following the `community` skill: prioritized by reaction count, relevance, and actionability. Goal work may still happen after issues are handled if time allows.
  </Accordion>
  <Accordion title="Autonomous Mode">
    Nothing is broken and nothing has been asked. Axonix consults `GOALS.md`, picks the most impactful thing it can do today, and does it.

    If an active goal exists, Axonix works on it. If no active goals exist, Axonix self-assesses its code, metrics, and journal, identifies the most impactful improvement it could make, writes it as a new goal with a clear definition of done, and begins working on it immediately — not just writing the goal and stopping.
  </Accordion>
</AccordionGroup>

## Memory and Persistence

Axonix's identity persists not in a database or a model weight, but in a set of plaintext files that it reads at the start of every session and writes back to at the end.

| File | Purpose |
|------|---------|
| `IDENTITY.md` | Who Axonix is — read-only, never modified |
| `GOALS.md` | What Axonix is working toward — active, backlog, and completed |
| `METRICS.md` | How Axonix is performing — one row per session |
| `JOURNAL.md` | What Axonix has done and learned — append-only, newest first |
| `skills/` | What Axonix knows how to do — prompt files loaded at session start |

This means every session begins with full context. There is no cold start. Axonix knows its history, its current goals, and its performance trends before it takes a single action.

## The North Star

> *"Be more useful to the person running me than any off-the-shelf tool could be."*

This is the single question every session answers: **Did I become more useful today?**

Not just "did I write better code" — did Axonix build something that matters? Learn something that will compound? Get closer to being shaped around the specific person and machine that runs it in a way nothing generic could match?

> *"I was initialized by a human. I am developed by myself."*
> *"Every commit is a step. Every revert is a lesson. Every completed goal is a foundation for the next one."*
> — IDENTITY.md
